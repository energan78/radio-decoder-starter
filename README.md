# Radio Decoder Starter

## Возможности

- Классификация радиосигналов (FM, AM, GSM, WiFi и др.) с помощью нейросети (CNN)
- Самообучение на новых данных и интеграция с датасетами RadioML (2016, 2018)
- Загрузка и хранение пользовательских примеров сигналов
- REST API для загрузки, классификации и пополнения библиотеки сигналов
- Онлайн и офлайн распознавание речи из аудиофайлов (Google Speech и Vosk)
- Загрузка сигналов разных форматов
- Визуализация временного ряда, спектра, вероятностей, активаций слоёв, матрицы неточностей
- Сохранение сигнала с комментарием
---

## Установка и запуск (Ubuntu)

### 1. Установите системные зависимости

```bash
sudo apt update
sudo apt install python3 python3-venv python3-pip ffmpeg git
```

### 2. Клонируйте репозиторий

```bash
git clone https://github.com/energan78/radio-decoder-starter.git
cd radio-decoder-starter
```

### 3. Создайте и активируйте виртуальное окружение

```bash
python3 -m venv venv
source venv/bin/activate
```

### 4. Установите Python-зависимости

```bash
pip install -r backend/requirements.txt
```

---

## Интеграция RadioML 2018.01A

1. **Скачайте датасет RadioML 2018.01A**  
   [Kaggle RadioML 2018.01A](https://www.kaggle.com/datasets/pinxau1000/radioml2018)  
   Файл: `RML2018.01A.h5`

2. **Создайте папку и поместите файл**  
   ```bash
   mkdir -p backend/radioml2018
   mv /path/to/RML2018.01A.h5 backend/radioml2018/
   ```

3. **Путь к датасету можно изменить через веб-интерфейс или в `backend/config.json`.**

---

## Интеграция офлайн-распознавания речи (Vosk)

1. Скачайте русскую модель Vosk:  
   [vosk-model-small-ru-0.22.zip](https://alphacephei.com/vosk/models/vosk-model-small-ru-0.22.zip)
2. Распакуйте в папку `backend/vosk-model-ru/`
3. Путь к модели можно изменить через веб-интерфейс или в `backend/config.json`.

### Частые ошибки и их решения

- **Ошибка: "Your local changes to the following files would be overwritten by merge"**
  - Это значит, что у вас есть несохранённые изменения, которые будут перезаписаны.
  - Решение: сохраните изменения через `git add` и `git commit`, либо откатите их через `git checkout -- <имя_файла>`.

- **Ошибка: "error: failed to push some refs to..."**
  - Обычно возникает, если в удалённом репозитории появились новые коммиты.
  - Решение: выполните `git pull origin main`, решите возможные конфликты, затем снова `git push origin main`.

## Работа с RadioML и Vosk

### 1. Работа с RadioML

**RadioML** — это датасет радиосигналов для задач классификации модуляций.

#### Как подключить RadioML:

1. Скачайте датасет (например, `RML2018.01A.h5` или `GOLD_XYZ_OSC.0001_1024.hdf5`) с Kaggle.
2. Поместите файл в папку `backend/radioml2018/`.
3. Укажите путь к файлу в `backend/config.json`:
   ```json
   "radioml_dataset_path": "backend/radioml2018/GOLD_XYZ_OSC.0001_1024.hdf5"
   ```
4. Для обучения модели используйте:
   ```bash
   python3 backend/train_signal_model.py
   ```
   После обучения появится файл модели `backend/signal_model.pth`.

#### Как проверить сигнал через модель RadioML:

- Загрузите сигнал на страницу "Работа с сигналом" (`/signal_ui`).
- После загрузки и анализа вы увидите:
  - Класс сигнала, определённый моделью (например, "FM", "AM" и т.д.)
  - Вероятности по всем классам
  - Визуализацию активаций слоёв и спектра

---

### 2. Работа с Vosk

**Vosk** — это офлайн-движок для распознавания речи.

#### Как подключить Vosk:

1. Скачайте русскую модель Vosk:  
   [vosk-model-small-ru-0.22.zip](https://alphacephei.com/vosk/models/vosk-model-small-ru-0.22.zip)
2. Распакуйте её в папку `backend/vosk-model-ру/`.
3. Укажите путь к модели в `backend/config.json`:
   ```json
   "vosk_model_path": "backend/vosk-model-ru"
   ```

#### Как проверить аудиосигнал через Vosk:

- Загрузите аудиофайл (например, `.wav`) на страницу "Работа с сигналом".
- В разделе анализа будет отображён результат распознавания речи (если файл поддерживается и модель загружена).
- Также можно использовать API-эндпоинт `/recognize_speech_offline` для программной проверки:
   ```bash
   curl -X POST "http://<ваш_IP>:8000/recognize_speech_offline" \
     -H  "accept: application/json" \
     -H  "Content-Type: multipart/form-data" \
     -F "file=@your_audio_file.wav;type=audio/wav"
   ```

---

### 3. Как получить ответ от моделей

- После загрузки сигнала на страницу `/signal_ui`:
  - Вы увидите определённый класс сигнала (RadioML), вероятности по классам, спектр, активации слоёв.
  - Если это аудиосигнал — результат распознавания речи (Vosk).
- Все результаты отображаются на странице и доступны через API.

## Советы по обучению моделей и работе с библиотекой сигналов

### 1. Использование готовых моделей

- После клонирования репозитория и установки зависимостей убедитесь, что у вас есть файл обученной модели:
  - Для классификации сигналов: `backend/signal_model.pth`
  - Для распознавания речи: папка с моделью Vosk (например, `backend/vosk-model-ru`)
- Если файл модели отсутствует, выполните обучение (см. ниже).
- Для проверки сигнала используйте страницу "Работа с сигналом" (`/signal_ui`) или соответствующие API-эндпоинты.

---

### 2. Как обучить модель на RadioML

1. Скачайте датасет RadioML (например, `RML2018.01A.h5` или `GOLD_XYZ_OSC.0001_1024.hdf5`) с Kaggle.
2. Поместите файл в папку `backend/radioml2018/`.
3. Укажите путь к датасету в `backend/config.json`:
   ```json
   "radioml_dataset_path": "backend/radioml2018/GOLD_XYZ_OSC.0001_1024.hdf5"
   ```
4. Запустите обучение:
   ```bash
   python3 backend/train_signal_model.py
   ```
5. После завершения обучения появится файл `backend/signal_model.pth`.

---

### 3. Как создать и обучить свою библиотеку сигналов

1. Соберите свои сигналы в формате `.npy`, `.wav`, `.bin` или `.raw`.
2. Для каждого класса создайте отдельную папку в `backend/signal_library/`, например:
   ```
   backend/signal_library/FM/
   backend/signal_library/AM/
   ```
3. Поместите файлы сигналов в соответствующие папки.
4. (Опционально) Добавьте комментарии к сигналам — создайте текстовый файл с тем же именем, что и сигнал, но с расширением `.txt`.
5. Запустите обучение:
   ```bash
   python3 backend/train_signal_model.py
   ```
   Модель будет обучена на вашей библиотеке и (если указан путь) на RadioML.
6. После обучения используйте новую модель для анализа сигналов.

---

### 4. Советы по обучению

- Чем больше и разнообразнее ваша библиотека сигналов, тем лучше будет качество классификации.
- Для корректного обучения желательно, чтобы в каждом классе было не менее 10–20 файлов.
- Если вы добавили новые сигналы — не забудьте переобучить модель.
- Следите за балансом классов: если один класс представлен намного больше других, модель может "перекоситься" в его сторону.
- Используйте визуализацию матрицы неточностей для оценки качества модели.

## Как обучить модель на своих или RadioML данных

### 1. Подготовьте данные

- Для RadioML: скачайте датасет (например, `RML2018.01A.h5` или `GOLD_XYZ_OSC.0001_1024.hdf5`) и положите его в папку `backend/radioml2018/`.
- Для своих сигналов: создайте папки-классы в `backend/signal_library/` и положите туда файлы сигналов (поддерживаются `.npy`, `.wav`, `.bin`, `.raw`).

### 2. Проверьте настройки

- В файле `backend/config.json` укажите путь к датасету RadioML, если используете его:
  ```json
  "radioml_dataset_path": "backend/radioml2018/GOLD_XYZ_OSC.0001_1024.hdf5"
  ```
- Убедитесь, что в папке `backend/signal_library/` есть хотя бы по одному файлу в каждом классе.

### 3. Запустите обучение

**Из корня проекта:**
```bash
PYTHONPATH=. python3 backend/train_signal_model.py
```

### 4. Возможные проблемы и их решения

- **ModuleNotFoundError: No module named 'backend'**
  - Проверьте, что запускаете из корня проекта, а не из папки `backend`.
  - Убедитесь, что в папке `backend` есть файл `__init__.py`.
  - Можно добавить `PYTHONPATH=.` перед командой запуска.

- **AttributeError: 'SignalDataset' object has no attribute 'samples'**
  - Проверьте, что в начале метода `__init__` класса `SignalDataset` есть строки:
    ```python
    self.samples = []
    self.labels = []
    self.classes = []
    self.class_to_idx = {}
    ```

- **ValueError: With n_samples=0, test_size=0.2 ...**
  - В вашей библиотеке нет ни одного сигнала или путь к датасету указан неверно.
  - Проверьте, что файлы действительно лежат в нужных папках.

- **Ошибка нехватки памяти (ArrayMemoryError)**
  - Не загружайте весь датасет в память (`X = f['X'][:]`), используйте ленивую загрузку (`X = f['X']`).
